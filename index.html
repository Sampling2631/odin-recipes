Odin Recipes

Purpose

This website is an example of (what I consider) proper HTML and CSS programming. And Javascript too, I suppose, which is to say that Javascript should be avoided. Use semantic HTML like in here folks. Run it through an HTML validator. Then write the CSS, with a proper color palette and some basic accessibility, like larger font sizes and 1.5x to 2x line spacing. Then run that through a CSS validator. To anyone coming here through the Odin Project, consider this a preview of things to come and habits to build. Also the sidebar on the recipes was completely unnecessary and was just for learning. Keep your websites simple. It's much less work.
